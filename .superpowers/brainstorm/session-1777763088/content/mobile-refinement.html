Refining Mobile "Scales"

Improving the general layout scaling for phone views

Current Issues

  • Hero Typography: Header font-size (4rem) might be too large for small screens, causing wrapping issues.
  • Mockup Scaling: Desktop frame is hard-scaled to 0.9, which might overflow or look disconnected.
  • Feature Grid: Single column layout is correct, but padding and spacing might feel "terrible" (too loose or too tight).
  • Mobile Frame: Currently hidden on mobile, which misses an opportunity for a native-feeling preview.
A

Fluid Typography

Use clamp() or more aggressive media queries to ensure text scales perfectly without wrapping.

B

Responsive Mockups

Replace hard scaling with container-aware layout. Enable the mobile-frame for phone views.

C

Refined Spacing

Tighten vertical gaps and horizontal padding to maximize screen real estate on mobile.
