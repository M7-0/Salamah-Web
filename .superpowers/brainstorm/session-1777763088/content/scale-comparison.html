Perfecting Mobile Sizing & Scale

Solving the "terrible" scale issues through mathematical and proportional refinement

What's wrong with the current scale?

The current 768px breakpoint uses "step-down" logic that isn't granular enough. A 4rem header on a 375px phone is ~50% of the screen width, which feels claustrophobic and "off-scale."

A

Fluid/Mathematical Scaling

Use clamp() for all typography and gaps. The site scales perfectly line-by-line as the phone gets smaller. No "jumps" between sizes.

B

Proportional Viewport Scaling

Scale the entire container using vw units for margins and padding. Maintains the "Desktop Look" but perfectly miniaturized for the phone.

C

Breakpoint-Surgical Scaling

Add specific overrides for 480px (typical phone) that aggressively reduce H1, stats, and feature card padding to reclaim white space.
